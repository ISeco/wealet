- **Auth con Google — bugs corregidos (2026-06-30)**: el login con Google devolvía 401 siempre. (1) Backend: `auth.service.ts` leía `tokenInfo.user_id` del endpoint tokeninfo de Google, pero ese campo ya no existe en la respuesta actual — el identificador viene en `tokenInfo.sub`. Corregido usando `sub`. (2) Frontend: tras un 200 exitoso, `AuthPage` seguía mostrando `/login` — el handler `onSuccess` de `useGoogleLogin` actualizaba el estado de auth pero nunca llamaba `navigate('/')`, a diferencia de `LoginForm`/`RegisterForm`. Corregido agregando `navigate('/', { replace: true })` tras `loginWithGoogle`.
- **Vercel — sesión se perdía al refrescar (2026-06-30)**: dos causas distintas, sin cambios de código necesarios para la primera.
  - (1) La API en Render tenía `NODE_ENV=development`, así que `auth.controller.ts` emitía la cookie `refresh_token` sin `secure`/`sameSite: 'none'`. Como el frontend (`wealet-web.vercel.app`) y la API (Render) son dominios distintos, el navegador descartaba la cookie en cada request cross-site, y el refresh tras recargar la página devolvía 401. Fix: setear `NODE_ENV=production` en las env vars del servicio de Render (no es un bug de código).
  - (2) Vercel devolvía 404 al refrescar en rutas no-raíz (ej. `/login`) porque no había `vercel.json` con rewrite para SPA routing — el GET directo a esa ruta no encontraba un archivo físico. Agregado `apps/web/vercel.json` con rewrite `/(.*) -> /index.html`. Pendiente: redeploy del frontend en Vercel para que tome efecto.
- **Fix: fechas futuras en transacciones/transferencias (2026-06-30)**: `DateInput` permitía seleccionar cualquier fecha sin tope. Frontend: se agregó prop `maxDate` (ISO) al componente — deshabilita días/meses futuros en el calendario y los marca visualmente; usado con la fecha de hoy en `TransactionFormModal`, `TransfersPage` y los filtros de rango de `TransactionsToolbar`. Backend: nuevo validador `@IsNotFutureDate()` en `common/validators/`, aplicado a `occurredOn` en `CreateTransactionDto`, `UpdateTransactionDto` y `CreateTransferDto` para cerrar el boundary ante llamadas directas a la API. No se tocó `ImportRowDto` (importación de datos históricos vía Excel).
- **Fix: CommandPalette se desbordaba de la pantalla (2026-06-30)**: con muchos fondos, la lista de resultados (Navegar a / Crear / Fondos recientes / Fondos) no tenía límite de alto ni scroll, y se cortaba contra el borde de la ventana. Estado vacío (sin búsqueda) ahora muestra solo "Acciones rápidas" (4 items: nueva transacción/transferencia/fondo/categoría, con ícono de color) más un mensaje "Escribe para buscar fondos, pantallas y otras opciones de la app.", fiel a `docs/design/screens/palette-quick.html`. Al escribir, se mantiene la búsqueda completa (nav + acciones + fondos) pero el contenedor de resultados ahora tiene `max-height` con scroll interno en vez de desbordar.
